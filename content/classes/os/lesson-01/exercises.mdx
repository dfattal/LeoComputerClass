# Lesson 1 — Exercises: The Frozen Computer

## How This Works

You'll write three functions that model a CPU running programs **one at a time** — and measure the damage when one program hogs it. Click **Run Tests** and the computer checks your timeline against the right answer.

**The model:**

- `programs` is a list of numbers. `programs[i]` = how many **ticks of work** program `i` needs.
- A **timeline** is a list where `timeline[t]` = which program ran on tick `t`.
- The CPU does exactly **one tick per moment**, in order.

## Tips If You're Stuck

1. **Timeline too short or too long?** Each program should appear exactly `work` times. Count them.
2. **`wait_time` off by a little?** Use `programs[:pid]` (up to but **not** including `pid`) — a program doesn't wait for itself.
3. **`longest_hog` returns 1 for everything?** Make sure you reset `run` to `1`, not `0`, when the id changes.
4. **`None` showing up?** Every function needs a `return`.
