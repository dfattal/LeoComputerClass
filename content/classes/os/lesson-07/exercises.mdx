# Lesson 7 — Exercises: The Need for Speed

## How This Works

You'll build an **LRU cache**. The cache is a list, **most-recently-used item first**. The item at the back is the one used longest ago — the first to go when the cache is full.

**The rules:**

- Using an item moves it to the **front** (it's the most recent now).
- If the cache grows past `capacity`, drop the item at the **back** (least recently used).
- An item appears in the cache **at most once**.

## Tips If You're Stuck

1. **An item appears twice in the cache?** Remove the old copy *before* inserting it at the front.
2. **Evicting the wrong item?** The back of the list is the LRU. Keep `new[:capacity]` (the front items), which drops the back.
3. **`total_cost` always counts a hit?** You must check the cache **before** calling `access` — otherwise the item you just added looks like a hit.
4. **Last example surprised you?** With capacity 2 and reads A, B, C, A — by the time A comes back, it was evicted to make room for C. That's **thrashing**: the cache is too small, so everything's a miss.
