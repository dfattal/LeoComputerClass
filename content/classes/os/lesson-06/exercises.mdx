# Lesson 6 — Exercises: Saving for Keeps

## How This Works

You'll build a tiny **file system**. The disk is a list of blocks: each block holds the **name** of the file that owns it, or `""` (empty string) if it's **free**.

**The rules:**

- `save` uses **first-fit**: the first run of `size` free blocks *in a row*.
- A file's blocks must be **contiguous** (next to each other).
- No room? Return the disk **unchanged**.
- Always return a **new** list.

## The Round-Trip Test

The real proof your file system works: **save a file, then look it up.**

```python
disk = save(["", "", "", ""], "cat", 2)   # ["cat", "cat", "", ""]
blocks_of(disk, "cat")                      # → [0, 1]  ✅ found exactly where we saved it
```

If save-then-find returns the blocks you just wrote, your file system is honest.

## Tips If You're Stuck

1. **`save` finds blocks that aren't in a row?** Reset your `run` to `0` whenever you pass an owned block.
2. **Comparing to the wrong "empty"?** A free block is the empty string `""`, not `0` and not `None`.
3. **`blocks_of` returns blocks in the wrong order?** `enumerate` gives them low-to-high already — don't sort or reverse.
4. **Delete wiping the wrong file?** Only blank blocks where `b == name`; leave every other name alone.
