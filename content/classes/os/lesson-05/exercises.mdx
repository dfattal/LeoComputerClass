# Lesson 5 — Exercises: Swiss-Cheese Memory

## How This Works

You'll write three tools that fight **fragmentation**. Memory is the same list as last lesson: `0` is a free slot, any other number is an owner id.

**Key idea:** a "free block" or "gap" is a run of `0`s **in a row**. Scattered zeros don't count as one block.

## Tips If You're Stuck

1. **`largest_free_block` counts scattered zeros as one block?** Reset your run to `0` on any non-zero slot.
2. **`best_fit` returns the first gap instead of the smallest?** Compare lengths — only switch when a *fitting* gap is **strictly smaller** than your current best.
3. **`best_fit` measures gaps wrong?** Make sure you advance past **all** the zeros in a gap before checking the next one (don't recount the same gap).
4. **`compact` changes the order of programs?** A list comprehension keeps them in order — don't sort.
