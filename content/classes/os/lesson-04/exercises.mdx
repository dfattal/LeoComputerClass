# Lesson 4 — Exercises: Whose Memory Is This?

## How This Works

You'll build a **memory allocator**. Memory is a list of slots: each value is the program id that owns that slot, or `0` for **free**.

**The rules:**

- `allocate` uses **first-fit**: the first run of `size` free slots *in a row*.
- A block must be **contiguous** (the free slots next to each other).
- No room? Return memory **unchanged**.
- Always return a **new** list — don't scribble on the one you were given.

## Tips If You're Stuck

1. **Allocator grabs slots that aren't in a row?** Reset your `run` counter to `0` every time you pass an owned slot.
2. **Filled the wrong slots?** Remember where the run **started**, not where it ended — fill from `start`.
3. **No-room case broken?** If you scan the whole list and never hit `size` free in a row, return the original memory.
4. **Your tests change the input list?** Build and return a fresh list; never edit `memory` in place.

Build all three and run the panel — you'll see colored blocks appear as programs allocate, and a gap open up when one frees. Next lesson: those gaps cause a sneaky problem called **fragmentation**.
