# Lesson 2 — Exercises: Taking Turns

## How This Works

You'll build a **round-robin scheduler**. Same model as last lesson — `programs[i]` is the ticks of work program `i` needs, and a **timeline** records who ran each tick — but now you rotate turns instead of running each program to the end.

**The rules:**

- Go around the programs in order, `0, 1, 2, ...`, then back to `0`.
- Each turn runs up to `quantum` ticks — but never more than the program has left: `min(quantum, remaining)`.
- Skip any program that's already finished.
- Keep going around until **everyone** is done.

## Tips If You're Stuck

1. **Infinite loop / browser hangs?** Your `while` never ends because you forgot to subtract from `remaining`. Each turn must shrink someone's work.
2. **A program runs too long?** Use `min(quantum, remaining[pid])` so its last turn doesn't overshoot.
3. **Timeline has the wrong total length?** It should equal the sum of all the work. Count your ticks.
4. **`finish_order` lists a program twice?** Only append a pid the single moment it hits 0 — not every turn.
