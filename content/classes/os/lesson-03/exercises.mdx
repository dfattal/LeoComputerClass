# Lesson 3 — Exercises: Fair vs. Fast

## How This Works

You'll build a **priority scheduler**. Each program has work left (`remaining[i]`) and a priority (`priorities[i]`). **Higher priority = more urgent.**

**The rules:**

- Only consider programs that still have work (`remaining[i] > 0`).
- Always pick the **most urgent** of those (biggest priority number).
- On a tie, pick the **lower** program id (so it's predictable — never random).

## Tips If You're Stuck

1. **`pick_next` returns a finished program?** Make sure you skip any id where `remaining[pid] <= 0`.
2. **Ties picking the wrong one?** Use a strict `>` (not `>=`) so the first/lower id keeps the lead.
3. **`priority_run` loops forever?** You must subtract 1 from `remaining[pid]` every tick.
4. **`starves` wrong on "never ran"?** Handle that case **after** the loop — returning `True`.
