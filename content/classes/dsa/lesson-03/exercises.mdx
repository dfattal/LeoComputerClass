## Your mission

Both functions get a list that is **already sorted** (smallest to biggest). Use that!

### Exercise 1 — `binary_search(sorted_arr, target)`

Keep two markers, `lo` and `hi`, for the part of the list still in play. Check the middle. Shrink the range by half each loop. Return the index when you land on `target`, or `-1` if `lo` ever passes `hi`.

```python
binary_search([1, 3, 5, 7, 9, 11], 7)   # -> 3
binary_search([1, 3, 5, 7, 9, 11], 1)   # -> 0
binary_search([1, 3, 5, 7, 9, 11], 4)   # -> -1  (not in the list)
binary_search([], 4)                    # -> -1
```

**Hint:** Start `lo = 0` and `hi = len(sorted_arr) - 1`. Loop `while lo <= hi:`. Compute `mid = (lo + hi) // 2`. If `sorted_arr[mid]` is too small, move `lo = mid + 1`; if too big, move `hi = mid - 1`.

### Exercise 2 — `binary_steps(sorted_arr, target)`

The exact same search, but count how many times you look at a middle box. Add `1` each loop, before you compare.

```python
binary_steps([1, 2, 3, 4, 5, 6, 7], 4)   # -> 1   (4 is the middle — one look!)
binary_steps([1, 2, 3, 4, 5, 6, 7], 1)   # -> 3   (a corner takes a few)
binary_steps([1, 2, 3, 4, 5, 6, 7], 8)   # -> 3   (missing — narrows to nothing)
```

**Hint:** Copy your binary search, add `steps = 0` before the loop and `steps += 1` as the first line inside it. Return `steps` instead of the index.

When both work, compare the two lines on the graph. The straight one is last lesson's linear search. Yours is the flat one hugging the bottom — **that's the win**, and you can prove it.
