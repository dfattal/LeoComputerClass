## How This Works

You'll turn your drag simulator into a measuring-and-searching tool. Use `g = 9.8`, keep **up positive**, `import math`, and use the **drag model from Lesson 3**:

```python
spd = math.sqrt(vx*vx + vy*vy)
ax  = -(k/m) * spd * vx
ay  = -g - (k/m) * spd * vy
```

Update velocities first, then positions. Answers are approximations — the tests accept anything **close enough**.

## Tips If You're Stuck

1. **`best_angle` returns `None`?** Make sure you set `best_angle_so_far = angle` inside the `if r > best_range:` block.
2. **`simulate_shot` list looks wrong?** Check you return `[x, peak, t]` in that order, and that `peak` starts at `0.0` and updates with `if y > peak`.
3. **Sweep feels slow?** That's normal — it runs 61 simulations! The bigger `dt = 0.005` in Exercise 2 keeps it quick.
4. **Range too big (like Lesson 2)?** Don't forget to update `vx` with drag — the dart slows sideways too.
