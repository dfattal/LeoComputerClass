## Don't store the password — store its fingerprint

When you make a password, a good website doesn't save the password itself. It runs the password through a **hash** — a scrambler that turns text into a number you can compute *forwards* but practically never run *backwards*.

Think of it like a fingerprint. Your fingerprint comes from *you*, but nobody can rebuild *you* from a fingerprint. Same idea:

```python
simple_hash("dragon")   # -> 183714459
simple_hash("leo")      # -> 315567
```

The site stores only that number. When you log in, it hashes what you typed and checks if the numbers match. So even if a thief steals the whole database, they get a pile of fingerprints — not passwords. 🎉

Here's a tiny real hash you'll build. It rolls through the letters, mixing each one into a running number:

```python
h = 7
for ch in password:
    h = (h * 31 + ord(ch)) % 1000000007
```

`ord(ch)` is the letter's number (you met it in Secret Codes), and `% 1000000007` keeps the number from growing forever.

## The crack: a dictionary attack 📖

There's a weakness. A hash is **the same every time** — `"dragon"` always makes `183714459`. So an attacker who steals the hashes can fight back: take a list of the most common passwords, hash every one, and look for a match.

```python
dictionary_attack(183714459, ["password", "123456", "dragon", "qwerty"])
# -> "dragon"   (cracked! it was a common word)
```

If your password is a common word, hashing doesn't save you — the attacker just hashes the whole dictionary. But a weird password like `xQ9!vzt` isn't in any list, so the attack comes up empty:

```python
dictionary_attack(simple_hash("xQ9!vzt"), ["password", "123456", "dragon"])
# -> ""   (not found — strong passwords win again)
```

## The defense: add salt 🧂

How do we protect the *common* passwords too? **Salt.** Before hashing, mix in a few extra letters that are different for every user:

```python
salted_hash("dragon", "a1")   # -> 419639183
salted_hash("dragon", "z9")   # -> 335016541   (same password, different hash!)
```

Now two people who both picked `"dragon"` get *different* hashes, and the attacker's giant pre-hashed dictionary is useless — they'd have to redo the whole thing for every single salt. That's why every real site salts its passwords.

## What you'll build

- **`simple_hash(password)`** — scramble text into a one-way number.
- **`dictionary_attack(target_hash, wordlist)`** — hash common words to crack a weak password.
- **`salted_hash(password, salt)`** — mix in salt so identical passwords hash differently.
