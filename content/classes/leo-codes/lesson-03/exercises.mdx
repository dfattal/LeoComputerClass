# Lesson 3 — Exercises: The Substitution Cipher

## How This Works

Your key is a **26-letter scrambled alphabet**. Position tells you everything: `key[0]` is what `a` becomes, `key[1]` is what `b` becomes, … `key[25]` is what `z` becomes.

The starter gives you `ALPHA = "abcdefghijklmnopqrstuvwxyz"` (the normal alphabet) and a sample scrambled `KEY` to test with. The same rules as always:

- Substitute letters `a`–`z`; leave spaces and other characters unchanged.
- The cipher letter for a character `c` is `key[ord(c) - 97]`.

## Tips If You're Stuck

1. **Reverse key looks scrambled wrong?** Remember you're placing `ALPHA[i]` into the slot named by `key[i]` — not the other way around. The cipher letter is the *address*, the plain letter is what goes there.
2. **Decrypt gives gibberish?** Make sure `unsubstitute` uses `invert_key(key)`, not `key` itself.
3. **Spaces vanished or changed?** Only substitute when `"a" <= c <= "z"`; otherwise keep the character.
4. **Want proof it works?** Print `unsubstitute(substitute("secret plans", KEY), KEY)` — it should say `secret plans`.

Build all three and you've got a code no one can brute-force. Next lesson: how to crack it anyway — by *counting letters.* Let's go!
