# Lesson 8 — Exercises: The Public Lock (RSA)

## How This Works

You're building a tiny **RSA** with small numbers so every step is visible. The starter gives you everything set up:

- Primes `p = 5`, `q = 11`
- `n = p * q = 55` (the public clock size)
- `phi = (p - 1) * (q - 1) = 40` (secret helper)
- A sample public exponent `e = 3`
- A `gcd(a, b)` helper (greatest common divisor)

Messages are **numbers smaller than `n`**. Lock with the public exponent `e`, unlock with the private key `d`. Use the fast three-argument `pow(base, exp, mod)` for the clock math.

## Putting It Together — Your Public Lock

Try the full system in the editor. The public key `(e, n)` goes to everyone; the private key `d` stays secret:

```python
e, n, phi = 3, 55, 40
d = private_key(e, phi)        # your SECRET private key
print("public key:", (e, n))   # hand this out freely
print("private key:", d)       # never share this!

# Your sister uses your PUBLIC key to lock the number 9 to you:
locked = encrypt(9, e, n)
print("she sends:", locked)

# Only YOU can unlock it, with your private key:
print("you read:", decrypt(locked, d, n))   # back to 9
```

## Tips If You're Stuck

1. **`private_key` runs forever?** Make sure your loop checks `(e * d) % phi == 1` (remainder of 1), and that you start `d` at `1` and add `1` each time.
2. **Round-trip gives the wrong number?** Lock with `e`, unlock with `d` — don't swap them. And keep your message smaller than `n` (55).
3. **Slow or giant numbers?** Use the three-argument `pow(base, exp, mod)`, not `(base ** exp) % mod`.
4. **Want the full picture?** Print the public key `(e, n)` and the private key `d`, then lock and unlock a number to watch it return home.
