# Lesson 6 — Exercises: The One-Time Pad

## How This Works

The key is now a **list of numbers**, the same length as the message — one key number per letter. You encrypt and decrypt by XOR-ing each letter with its **matching** key number, walking both lists together. Python's `zip(a, b)` pairs them up for you:

```python
for c, k in zip(message, key_nums):
    ...   # c is a character, k is its key number
```

As before: `ord(c)` turns a letter into a number, `chr(n)` turns a number back into a letter, and `^` is XOR.

## Tips If You're Stuck

1. **Lists different lengths?** `zip` stops at the shorter one — for the pad, the key should be as long as the message.
2. **Decrypt gives gibberish?** Use the **same** key list, in the **same order**, that you encrypted with.
3. **Mixing up encrypt vs the leak?** `otp_encrypt` pairs a *message* with a *key*. `two_time_leak` pairs two *ciphertexts* with each other — no key needed, which is exactly the danger.
4. **Want proof?** Print `otp_decrypt(otp_encrypt("fire now", [7, 7, 7, 7, 7, 7, 7, 7]), [7, 7, 7, 7, 7, 7, 7, 7])`.
