# Lesson 7 — Exercises: Key Exchange by Color-Mixing

## How This Works

Everything happens on a "clock" with a prime number of hours `p`: do the math, then take the remainder with mod `p`. Python has a built-in that does "raise to a power, then wrap mod p" in one fast, exact step:

```python
pow(base, exp, mod)    # = (base ** exp) % mod, but fast and exact
```

We'll use small numbers so you can check the clock math by hand if you want. The agreed-on public numbers in the examples are base `g = 5` and prime `p = 23`.

## Putting It Together

Try the whole handshake in the editor:

```python

alice_secret, bob_secret = 6, 15

A = public_key(g, alice_secret, p)   # Alice sends A
B = public_key(g, bob_secret, p)     # Bob sends B

alice_shared = shared_key(B, alice_secret, p)
bob_shared = shared_key(A, bob_secret, p)
print(alice_shared, bob_shared)      # same number!
```

## Tips If You're Stuck

1. **Numbers too big / slow?** Use the **three-argument** `pow(base, exp, mod)`, not `(base ** exp) % mod`. The three-argument form stays small and fast.
2. **Shared keys don't match?** Make sure `shared_key` raises the **received** mix to **your** secret — Alice uses Bob's `B` with her `a`, Bob uses Alice's `A` with his `b`.
3. **Forgot the mod?** Every step wraps mod `p`. Leaving it off gives a giant number instead of a clock value.

You can now agree on a key in front of everyone. Last lesson — the grand finale: a **public lock** anyone can close, that only *your* private key opens. Let's go!
