# Lesson 4 — Exercises: Frequency Analysis

## How This Works

You'll crack a substitution cipher by **counting letters**. The starter gives you:

- `ALPHA = "abcdefghijklmnopqrstuvwxyz"` — the normal alphabet.
- `ENGLISH_FREQ_ORDER = "etaoinshrdlcumwfgypbvkjxqz"` — English letters from **most** common to **least** common.

The rule stays the same: only letters `a`–`z` count; ignore spaces and other characters.

## Tips If You're Stuck

1. **Counts in the wrong slots?** The slot for a letter is `ord(c) - 97` (so `a`→0, `z`→25).
2. **Ties picking the wrong letter?** Use strict `>` when scanning for the max, and `(-counts[i], i)` as the sort key — the `i` breaks ties alphabetically.
3. **`KeyError` while translating?** Build the mapping for **all 26** letters (loop `k` over `range(26)`), so every possible cipher letter has a guess.
4. **It's not fully readable** — that's expected! Frequency analysis gets the common letters right; the rare ones need a human to finish.
